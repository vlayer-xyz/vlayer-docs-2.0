## ZK Prover Server

ZK Prover Server provides a REST API for compressing Web Proofs into zero-knowledge proofs (ZKPs). The API is available in two versions:

### v2 (recommended)

* [POST /api/v2.0/evm/compress-web-proof](/blockchain/rest-api/v2/compress-web-proof) accepts a Web Proof (output of `/prove` from [Web Prover Server](/server-side/rest-api/v2/prove) or payload from vouch webhook), verifies the Web Proof, parses HTTP transcript and returns a ZK proof with ABI-encoded public outputs (journalDataAbi)
* [GET /api/v2.0/evm/guest-id](/blockchain/rest-api/v2/guest-id) returns the ID of RISC Zero guest program used to generate ZK Proofs in /compress-web-proof endpoint
* [POST /api/v2.0/debug/evm/journal-decode-helper](/blockchain/rest-api/v2/journal-decode-helper) returns Solidity decode helpers for the journal data
* Fake-mode variants at `/api/v2.0/fake/evm/...` for development and testing without real ZK proof generation

See the [v0 vs v2 changelog](/blockchain/rest-api/v2/changelog) for migration details.

### v0

* [POST /api/v0/compress-web-proof](/blockchain/rest-api/v0/compress-web-proof) — same as v2 but allows anonymous and public client ID access
* [GET /api/v0/guest-id](/blockchain/rest-api/v0/guest-id) — same as v2 without `apiVersion` in response

## Example Use Cases

Below are some common use cases of ZK Prover Server:

* Uploading Web Proof on-chain - ZK proof provides succinctness and ease of on-chain verification compared to raw Web Proofs.
* Hiding Web Proof as private input - ZK proof allows you to prove the existence and validity of Web Proof data without revealing the actual Web Proof content.

### Real-World Examples

Here are concrete examples of what you can build with ZK Prover Server:

* **Prove your Coinbase Total Balance** - Verify your total portfolio value from Coinbase API and store it on-chain
* **Prove your Binance Trading Volume** - Demonstrate trading activity for DeFi protocols requiring volume verification
* **Prove ownership of Real World Assets (RWA)** - Verify property ownership, vehicle registration, or asset titles and tokenize them on-chain
* **Government ID Verification** - Prove you meet age or residency requirements without revealing personal details
* **Professional Licenses** - Verify professional certifications (medical, legal, financial) on-chain
* **Social Media Following** - Prove your follower count or engagement metrics for creator platforms
* **GitHub Contributions** - Verify open-source contributions for developer reputation systems
* **Domain Ownership** - Prove ownership of web domains for decentralized identity
